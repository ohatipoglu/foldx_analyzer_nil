#  GÖREV 1: README.md KULLANICI DOSTU YENİDEN DÜZENLEME
Mevcut proje yapısını (`core/`, `ui/`, `utils/`, `output/`), Biomatik filtreleri, batch/tekil modu ve modern UI özelliklerini göz önünde bulundurarak, hem bilim insanları (son kullanıcı) hem de geliştiriciler için anlaşılır, profesyonel bir `README.md` içeriği üret.

İçerik yapısı şu şekilde olmalı:
1. 🧬 Proje Başlığı & Kısa Açıklama (1-2 cümle, ne işe yaradığı)
2. ✨ Temel Özellikler (Madde madde, ikonlu, teknik ve kullanıcı odaklı)
3. ️ Ekran Görüntüsü Alanı (Placeholder markdown resmi)
4. 🚀 Hızlı Başlangıç (Kurulum, `pip install -r requirements.txt`, çalıştırma)
5. 🖥️ Kullanım Kılavuzu (GUI adımları: dosya seçme, batch modu, Biomatik aktivasyonu, sonuçları okuma)
6. 📂 Çıktı Dizini (`output/excels`, `output/plots` nasıl çalışır?)
7. 🧪 Bilimsel Arka Plan (ΔΔG eşikleri, outlier temizliği, Biomatik kurallar tablosu)
8. 🏗️ Mimari & Katkıda Bulunma (Modül yapısı, PR gönderme adımları)
9. 🐛 Sorun Giderme (Excel açık hatası, grafik görünmeme, encoding uyarıları)
10. 📜 Lisans & İletişim
✅ Markdown standartlarına uygun, okunabilir, Türkçe, emoji/ikon dengeli ve kopyalanabilir formatta olsun.

#  GÖREV 2: .gitignore DOSYASI OLUŞTURMA
Projeyi GitHub'a yüklerken gereksiz, kişisel veya otomatik üretilen dosyaların pushlanmasını engelleyecek standart ve projeye özel `.gitignore` içeriği üret.

Şunları KESİNLİKLE içermeli:
- Python artefact'ları: `__pycache__/`, `*.py[cod]`, `*.egg-info/`, `dist/`, `build/`
- Ortam & Bağımlılıklar: `venv/`, `.venv/`, `env/`, `*.log`
- IDE & Editör: `.vscode/`, `.idea/`, `*.swp`, `*.swo`
- İşletim Sistemi: `.DS_Store`, `Thumbs.db`, `Desktop.ini`
- Proje Özel: `output/` (tüm excel ve plot çıktıları), `*.fxout` (ham veri gizliliği/boyutu), `.env`, `config.local.py`
✅ Yorum satırları ile her bölüm ne için görmezden geliniyor açıkça belirtilsin. Standart GitHub Python `.gitignore` kurallarına uygun olsun.

#  BEKLENEN ÇIKTI SIRALAMASI
1. `README.md` için tam içerik (markdown code block içinde)
2. `.gitignore` için tam içerik (plaintext code block içinde)

4. Mevcut dosyalara dokunulmadığı, sadece içerik üretildiği teyit edilsin.

Lütfen talimatlara harfiyen uy. Sadece içerikleri üret, dosya silme/değiştirme işlemi yapma. Çıktıları net, profesyonel ve GitHub-ready formatta sun.
